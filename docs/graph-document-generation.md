# Graph Document Generation — Setup & Replication Guide

## Overview

DAX Document Generator creates Word documents and uploads them to SharePoint
via the Microsoft Graph API. It runs as an n8n workflow using client credentials
(app-only auth) — no user interaction required.

This is the foundation for automated report generation (Wealthbox reports,
compliance workpapers, client onboarding docs, etc.).

**Architecture:**
```
n8n webhook request
  -> Generate DOCX (docx npm package)
  -> Authenticate (client credentials → Azure AD token)
  -> Upload to SharePoint (Graph API PUT)
  -> Return SharePoint URL
```

## 1. Register the Entra App

Run the deployment script from the DAX repo root:

```powershell
# Ensure you're logged into the correct tenant
az login --tenant d2a3c346-00f3-47dd-a53e-caa3fca74714

# Register the app and store credentials in Key Vault
./scripts/Deploy-DocGenApp.ps1 -ClientName dakona-pilot
```

**What this does:**
- Creates "DAX Document Generator" app registration
- Grants **Sites.ReadWrite.All** + **Files.ReadWrite.All** (application permissions)
- Generates a client secret (365-day expiry)
- Grants admin consent automatically
- Stores `docgen-client-id` and `docgen-client-secret` in `kvdaxdakonapilot`

**Verify in Azure Portal:**
Entra ID → App registrations → DAX Document Generator → API permissions
→ All permissions should show "Granted for Dakona"

## 2. Find Your SharePoint Site ID

The n8n workflow needs the SharePoint site ID. Get it via Graph Explorer
or Azure CLI:

```bash
# Using the Graph API directly (after getting a token)
# Format: https://graph.microsoft.com/v1.0/sites/{domain}:/sites/{site-path}
# Example for Dakona's default team site:
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://graph.microsoft.com/v1.0/sites/dakona.sharepoint.com:/sites/DAX" \
  | jq '.id'

# Or for the root site:
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://graph.microsoft.com/v1.0/sites/dakona.sharepoint.com" \
  | jq '.id'
```

The site ID looks like: `dakona.sharepoint.com,<guid>,<guid>`

Save this — you'll need it for n8n configuration.

## 3. Configure n8n

### Prerequisites

n8n must have the `docx` npm package available. For self-hosted n8n
(Azure Container Apps), either:

**Option A — Custom Dockerfile (recommended):**
```dockerfile
FROM n8nio/n8n:latest
USER root
RUN npm install -g docx
USER node
```

**Option B — Environment variable:**
```
NODE_FUNCTION_ALLOW_EXTERNAL=docx
```
Then install `docx` in the n8n container's node_modules.

### Environment Variables

Set these on the n8n Container App:

| Variable | Value | Source |
|----------|-------|--------|
| `GRAPH_TENANT_ID` | `d2a3c346-00f3-47dd-a53e-caa3fca74714` | Client's Entra tenant ID |
| `GRAPH_CLIENT_ID` | From Key Vault | `docgen-client-id` secret |
| `GRAPH_CLIENT_SECRET` | From Key Vault | `docgen-client-secret` secret |
| `GRAPH_SITE_ID` | From step 2 | SharePoint site ID |
| `NODE_FUNCTION_ALLOW_EXTERNAL` | `docx` | Required for Code node |

### Import the Workflow

1. Open n8n UI
2. Go to **Workflows → Import from File**
3. Select `n8n/workflows/document-generator.json`
4. Activate the workflow

## 5. Replicating Per Client Tenant

To add document generation for a new client:

### Step 1: Register the Entra app in the client tenant

```powershell
# Log into the client's tenant
az login --tenant <client-tenant-id>

# Run the script with the client name
./scripts/Deploy-DocGenApp.ps1 -ClientName <client-name>
```

This stores `docgen-client-id` and `docgen-client-secret` in the
client's Key Vault (`kvdax<clientname>`).

### Step 2: Find the client's SharePoint site ID

```bash
# Get a token for the client tenant
TOKEN=$(curl -s -X POST \
  "https://login.microsoftonline.com/<client-tenant-id>/oauth2/v2.0/token" \
  -d "client_id=<docgen-client-id>&client_secret=<docgen-client-secret>&scope=https://graph.microsoft.com/.default&grant_type=client_credentials" \
  | jq -r '.access_token')

# Get the site ID
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://graph.microsoft.com/v1.0/sites/<client>.sharepoint.com:/sites/<site-name>" \
  | jq '.id'
```

### Step 3: Configure n8n environment variables

Set `GRAPH_TENANT_ID`, `GRAPH_CLIENT_ID`, `GRAPH_CLIENT_SECRET`, and
`GRAPH_SITE_ID` on the client's n8n Container App (using Key Vault
references where possible).

### Step 4: Import the workflow

Import `document-generator.json` into the client's n8n instance and
activate it.

## Webhook API Reference

### POST /webhook/generate-document

**Request body:**
```json
{
  "title": "Document Title",
  "content": "Document content with basic markdown:\n## Headings\n- Bullet points\nPlain text paragraphs",
  "templateType": "report",
  "folderPath": "General"
}
```

| Field | Required | Default | Description |
|-------|----------|---------|-------------|
| `title` | Yes | — | Document title (used as filename and H1 heading) |
| `content` | Yes | — | Document body (supports `##`, `###` headings, `- ` bullets, plain text) |
| `templateType` | No | `report` | Reserved for future template variants |
| `folderPath` | No | `General` | SharePoint folder path (relative to document library root) |

**Response:**
```json
{
  "success": true,
  "webUrl": "https://....sharepoint.com/.../Document-Title.docx",
  "fileName": "Document-Title.docx",
  "driveItemId": "graph-drive-item-id",
  "size": 12345
}
```

## Troubleshooting

**"AADSTS700016: Application not found"**
- The app isn't registered in the target tenant, or `GRAPH_TENANT_ID` is wrong.

**"AADSTS7000215: Invalid client secret"**
- Secret expired or doesn't match. Regenerate via Azure Portal or rerun
  `Deploy-DocGenApp.ps1`.

**403 on SharePoint upload**
- Admin consent not granted. Check Entra ID → App registrations → API permissions.
- The `GRAPH_SITE_ID` may be wrong — verify with Graph Explorer.

**"Cannot find module 'docx'"**
- The `docx` npm package isn't installed or `NODE_FUNCTION_ALLOW_EXTERNAL`
  isn't set. See section 3 prerequisites.

**Empty document / missing content**
- Check that `content` uses `\n` for line breaks (not `\\n` literals).
- The Code node supports `##` headings, `- ` bullets, and plain paragraphs.
