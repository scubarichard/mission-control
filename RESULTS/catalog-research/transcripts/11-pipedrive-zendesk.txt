Hello, this is a demonstration of my Pipe Drive to Zendesk automation that synchronizes deals that are completed or put in a specific status in Pipe Drive, and then synchronizes both the company organization in Zendesk, along with the contacts associated with that particular deal. Information once it's put in a specific state in Pipe Drive also exists within Zendesk along with the contact information so that we can begin issuing and addressing trouble tickets using Zendesk as a ticket management system. The first step is it waits for a deal to go to a specific status, in this case, implementation complete. Once it does that, we capture the information for that particular deal, query it, and search to see if that same deal exists within Zendesk by the deal, I mean the organization. If it does exist, we record what that organization ID is, and if it doesn't exist, we create that organization again, recording what the organization ID is, so that we can search whether or not the contact for that organization exists within Zendesk. If it does exist, we update some key information, maybe put a node in place that we received and synchronize the information. If it doesn't exist, we create that particular contact. In this case, it has two or three different contacts for the same organization, we enter in and create those contacts within Zendesk. The use case of this is that we always keep Pipe Drive, after it reaches a certain deal flow scenario, synchronized with an appropriate company in Zendesk along with its contacts.
